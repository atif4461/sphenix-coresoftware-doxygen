Detailed Description

BcoRangeCheck module verifies BCO clock continuity across events It inherits from SubsysReco and integrates with the Fun4All framework The module initializes resources processes each event to track BCO values and finalizes at the end of the job Users can set the event node name via MyEvtNode default node is INTTRAWHIT Internally it records seen BCO clocks counts differences and stores them in associative containers for later analysis This comment was generated by openai/gpt-oss-120b at temperature 0.1.
