Detailed Description

mvtx raw data decoder

Decodes raw MVTX detector data and fills hit set containers for offline reconstruction Supports optional offline pixel masking configurable via hot pixel mask Allows reading strip width from conditions database and setting strobe width in microseconds Can operate in triggered mode and uses configurable node names for raw hit and event header data

run initialization

Initialize run specific objects and configuration for the MVTX decoder.

This method creates required nodes, validates the presence of the raw event header and raw hit container, checks for obsolete header versions, retrieves the run number from reconstruction constants, optionally reads the strobe width from the conditions database, applies a default value if necessary, stores the strobe width back into the constants, and loads the hot pixel mask when offline masking is enabled.

Parameters
topNodepointer to the top PHCompositeNode of the current Fun4All job.
Returns

overridevirtual

event processing

Process a single event for MVTX combined raw data.

Parameters
topNodepointer to the top node of the Fun4All node tree
Returns
integer status code indicating event processing result

The method retrieves required nodes from the top node extracts the level 1 bunch crossing identifier from the GL1 raw hit synchronises the BCO values with the MVTX event header iterates over all raw hits in the MVTX raw hit container determines the strobe index by locating the hit BCO in the combined strobe list skips hits with strobe index outside the allowed signed 5 bit range builds a hit set key from layer stave chip and strobe index registers the hit set key with the helper object obtains or creates the corresponding hit set in the hit set container creates a hit key from column and row checks for duplicate hits and optionally applies an offline hot pixel mask creates a new TrkrHitv2 object and inserts it into the hit set finally returns the standard EVENT_OK code
