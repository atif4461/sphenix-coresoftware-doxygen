Record the translation,boost,rotation for HepMC frame to lab frame according to collision settings.

move vertex in translation,boost,rotation according to vertex settings

Transform a HepMC generated event into the laboratory frame by applying vertex translation, boost, and rotation.

The function operates in two modes:

  • Vertex reuse: If _reuse_vertex is enabled, the collision vertex, boost vector, rotation axis, and rotation angle are copied from a previously stored sub‑event identified by _reuse_vertex_embedding_id.
  • Full transformation: Otherwise the collision vertex is generated (optionally with beam‑bunch interaction), beam directions are built from configured theta/phi pairs, angular divergences are smeared, and the resulting boost vector (half‑sum of the two beam vectors) and rotation (aligning the beam‑difference axis with the global z‑axis) are stored in the event.

Diagnostic output is produced when m_verbosity is non‑zero.

Set the beam angular Z coefficients for horizontal and vertical components.

Central beam angle shift as linear function of longitudinal vertex position, d_shift/dz, which is used to represent leading order effect of crab cavity momentum kick on the beam bunch

Parameters

[in]beamA_vbeamA, vertical angle dv/dz. BeamA is aimed to +z direction in the HepMC event generator's coordinate

[in]beamB_vbeamB, vertical angle dv/dz. BeamA is aimed to -z direction in the HepMC event generator's coordinate

This method stores the provided angular coefficients for two beams (A and B) in the internal matrix m_beam_angular_z_coefficient_hv. The coefficients are organized as {{beamA_h, beamA_v}, {beamB_h, beamB_v}} where the first pair corresponds to beam A and the second pair to beam B.

toss a new vertex according to a Uniform or Gaus distribution

Configure the vertex distribution functions for the x, y, z, and t coordinates.

This method assigns the user‑specified distribution functions to the internal helpers that generate the primary interaction vertex. It aborts if the beam‑bunch simulation mode is enabled, because in that mode the vertex is derived from the simulated bunch interaction rather than from explicit distributions.
