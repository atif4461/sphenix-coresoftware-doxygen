Detailed Description

micromegas clusterizer

MicromegasClusterizer provides clustering of Micromegas detector hits in the Fun4All reconstruction chain The module derives from SubsysReco and implements Init InitRun process_event and End lifecycle callbacks User configurable parameters include default pedestal value usage flag calibration file name and option to drop single strip clusters Internal data structures store calibration data and a map of hitset identifiers to cluster counts This comment was generated by openai/gpt-oss-120b at temperature 0.1.

int MicromegasClusterizer::process_event ( PHCompositeNode * topNode)
overridevirtual

event processing

Process a single event to form Micromegas clusters from hitsets.

This method retrieves the Micromegas geometry, hitset container, cluster container, cluster‑hit association, and Acts geometry from the node tree. It iterates over all Micromegas hitsets, sorts hits by strip number, groups adjacent strips into clusters, computes cluster charge, position, and error using the detector segmentation, pitch, and strip length, and stores the resulting clusters in the TrkrClusterContainer while updating the TrkrClusterHitAssoc map and internal cluster counters.

Parameters
topNodePointer to the top PHCompositeNode of the current event containing all required sub‑nodes (geometry, hitsets, clusters, associations, Acts geometry).
Returns
