Detailed Description

2D magnetic field map implementation for sPHENIX simulations

Loads field components from a text file and provides lookup in Cartesian and cylindrical coordinates. The constructor requires a filename, optional verbosity level and optional magnetic field rescaling factor.

◆ GetFieldCyl_nocache()

void PHField2D::GetFieldCyl_nocache ( const double CylPoint[4],
double * BfieldCyl ) const

Compute the magnetic field vector in cylindrical coordinates at a given point without using cached values.

This routine performs a bilinear interpolation of the field map stored in the PHField2D object. The input point is specified by its axial (z) and radial (r) coordinates contained in the first two elements of CylPoint. The method checks that the point lies within the defined z and r ranges; if the point is outside the map the field components are left zero. Verbosity levels greater than two produce diagnostic output to std::cout.

The interpolated field components are written to the three‑element array BfieldCyl in the order (Bz, Br, Bphi). The phi component is always set to zero because the field map is assumed to be azimuthally symmetric.

Parameters
CylPointArray of four doubles where CylPoint[0] is the axial coordinate z and CylPoint[1] is the radial coordinate r (the remaining entries are ignored by this routine).
BfieldCylOutput array of three doubles that receives the interpolated magnetic field components (Bz, Br, Bphi) in the same units as the field map (gauss).
Returns
None (void). The result is stored in BfieldCyl.

Compute the magnetic field vector at a Cartesian point using the internal cylindrical field map.

access field value Follow the convention of G4ElectroMagneticField

Parameters
[in]Pointspace time coordinate. x, y, z, t in Geant4/CLHEP units
[out]Bfieldfield value. In the case of magnetic field, the order is Bx, By, Bz in in Geant4/CLHEP units

The routine converts the input Cartesian coordinates (x, y, z) to cylindrical coordinates (z, r, phi), queries the field in cylindrical representation via GetFieldCyl, and then transforms the result back to Cartesian components. If the point lies outside the configured z‑range, the field is set to zero. Verbose output is produced when the object's verbosity level exceeds 2.

Parameters
pointInput array of four doubles; point[0]=x, point[1]=y, point[2]=z, point[3] unused.
BfieldOutput array of three doubles that will contain the field components Bfield[0]=Bx, Bfield[1]=By, Bfield[2]=Bz after execution.
Returns
None.
