◆ initializeBranches()

void KFParticle_nTuple::initializeBranches ( PHCompositeNode * topNode)

Initialises required branches based off the user selection (number of tracks, PV constraints etc ) and sets branch names if specified.

Initialize the ROOT nTuple tree and define all branches for KFParticle output.

This method creates a new TTree named DecayTree (or replaces an existing one) and registers branches that store the calculated kinematic and vertex information for the mother particle, any intermediate states, and all daughter tracks. Branch names are sanitized to avoid illegal characters (e.g. '/', '(', ')', '+', '-', '*') by replacing them with underscores or descriptive strings. Optional branches are added depending on configuration flags:

  • m_constrain_to_vertex_nTuple – adds impact‑parameter, decay‑time, and vertex‑related branches.
  • m_get_all_PVs – adds per‑primary‑vertex impact‑parameter branches.
  • m_has_intermediates_nTuple – creates branches for each intermediate state.
  • m_get_dEdx – adds dE/dx information for daughter tracks.
  • m_calo_info, m_truth_matching, m_detector_info – invoke helper functions to add calorimeter, truth‑matching, and detector‑specific branches.
  • m_get_trigger_info – builds trigger‑related branches via buildTriggerBranches().

The method also defines pairwise daughter‑daughter distance‑of‑closest‑approach (DCA) branches and primary‑vertex parameters when vertex constraints are enabled.

Parameters
topNodePointer to the top PHCompositeNode of the current Fun4All node tree. Used only for building trigger branches; the tree itself is not modified.
Returns
