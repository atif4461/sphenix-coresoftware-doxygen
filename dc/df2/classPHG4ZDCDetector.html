Detailed Description

Geometry and hit handling for the Zero Degree Calorimeter (ZDC) subsystem.

This detector class derives from PHG4Detector and is responsible for constructing the ZDC geometry within the world volume, managing GDML export configuration, and providing accessors for layer identification and super‑detector naming.

Construction

Parameters
subsysPointer to the owning PHG4Subsystem.
nodePointer to the top level PHCompositeNode where detector data are stored.
parametersPointer to PHParameters containing ZDC configuration values.
dnamDetector name used for node registration.
detidInteger identifier for the ZDC instance.

Geometry construction

The ConstructMe method builds the full detector geometry. Internally it calls ConstructTower to create individual tower volumes of a given type.

Volume classification

Logical volumes are stored in sets according to their material role: m_AbsorberLogicalVolSet – absorber volumes, m_ScintiLogicalVolSet – scintillator volumes, m_FiberLogicalVolSet – fiber volumes, m_SupportLogicalVolSet – support structures.

Accessors

IsInZDC returns a non‑zero identifier when a physical volume belongs to the ZDC. SuperDetector sets or retrieves the name of the super‑detector grouping. get_Layer returns the current layer index. GetDisplayAction provides the associated PHG4ZDCDisplayAction instance.

Configuration parameters

Geometry dimensions such as plate thicknesses, absorber sizes, fiber dimensions, hole radii, window sizes and module counts are read from the PHParameters object.

Note
Ownership of objects placed on the node tree is transferred to the Fun4All framework; the detector does not delete them.
