Detailed Description

Class to write C binary file.

Mille class handles alignment output records Constructor opens output file optionally in binary mode and optionally writes zero entries Destructor closes the output file mille method records a measurement given numbers of local and global derivatives arrays labels measurement value and sigma special method records auxiliary integer and floating data for the current record kill method aborts the current record without writing end method finalizes the output file newSet prepares internal buffers for a new record checkBufferSize verifies that buffers can hold the requested numbers of local and global entries myOutFile stores the output stream myAsBinary selects binary or text output myWriteZero controls writing of zero valued derivatives or labels myBufferSize defines the size of internal integer and float buffers myBufferInt stores integer labels and other integer data myBufferFloat stores floating point derivative values myBufferPos tracks the current position in the buffers myHasSpecial indicates whether special data have been supplied for the current record myMaxLabel defines the largest allowed label value This comment was generated by openai/gpt-oss-120b at temperature 0.1.

Class to write a C binary (cf. below) file of a given name and to fill it with information used as input to pede. Use its member functions mille(), special(), kill() and end() as you would use the fortran MILLE and its entry points MILLSP, KILLE and ENDLE.

For debugging purposes constructor flags enable switching to text output and/or to write also derivatives and labels which are ==0. But note that pede will not be able to read text output and has not been tested with derivatives/labels ==0.

◆ end()

void Mille::end ( )

Write buffer (set of derivatives with same local parameters) to file.

Finalize the current buffer and write its contents to the output file.

The method checks whether any entries have been stored in the internal buffer. If data are present, it computes the total number of words to write and serializes the buffer either in binary or in ASCII format depending on the myAsBinary flag. After writing the buffer is reset to an empty state for the next set of derivatives. This comment was generated by openai/gpt-oss-120b at temperature 0.1.
