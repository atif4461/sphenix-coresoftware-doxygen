Detailed Description

Class SingleTriggeredInput provides input handling for single trigger events within the Fun4All framework It inherits from Fun4AllBase and InputFileHandler and manages a pool of events read from files The class maintains event iterators buffers and clock alignment checks for detector packets It creates DST nodes fills event vectors and supports packet keeping and alignment problem handling Configuration includes run number event number and flags for packet retention Internal data structures store event deques packet maps clock arrays and alignment counters

Packet * pkt,

virtual

Fill internal clock and clock‑difference arrays for a given packet.

This routine extracts the BCO (bunch crossing) clock from the supplied Event for the packet identified by pkt and stores it in the per‑packet clock array. It also computes the difference to the previous clock value and stores that in the corresponding difference array. Special handling is applied when the packet clocks have been copied from a FEM source; in that case the clock is taken from the previous event in the packet deque. The function also updates GL1 specific bookkeeping when the object is a SingleGl1TriggeredInput.

Parameters
evtPointer to the current Event object from which the clock is read.
pktPointer to the Packet object whose identifier determines the map key.
event_indexZero‑based index of the event within the packet pool.
Returns
None.

◆ FillPool()

void SingleTriggeredInput::FillPool ( )
virtual

Fill the event pool for a SingleTriggeredInput instance.

This method attempts to combine events from multiple SEB packets into a synchronized pool. It first checks for global termination conditions (AllDone or EventAlignmentProblem) and aborts if any are true. It then verifies that not all packets are marked as bad; if they are, the event alignment problem flag is set and processing stops.

When input files are not exhausted, the method fills the internal event vector and validates the GL1 clock map. It clears any previous clock overrides and extracts the GL1 packet identifier and its clock‑difference array. If the GL1 differences consist solely of the maximum 64‑bit value, the current pool is skipped.

For each SEB packet the method checks the size of the packet pool, skips packets already flagged with alignment problems, and determines whether the pool is aligned using CheckPoolAlignment. If alignment is achieved, bad indices are recorded, clock differences are printed for debugging, and any required shift (‑1 or +1) is applied to the packet clocks and event deque. Shift handling includes updating clock arrays, recomputing differences, inserting or dropping dummy events, and adjusting the packet‑shift offset.

If alignment fails, the method logs the failure, prints the GL1 versus SEB clock values, increments the alignment‑failure counter, marks the entire pool for ditching, and, after exceeding the maximum retry limit, permanently flags the packet as having an alignment problem.

The function returns void.
