Detailed Description

PHParameterInterface provides an abstract base for handling analysis parameters within the Fun4All framework It stores parameter values of type double int and string and supports default values and node tree persistence The class disables copy construction and assignment to avoid pointer ownership issues Derived classes must implement SetDefaultParameters to define their own default parameter set Public methods allow setting and retrieving parameters by name as well as updating from macro configuration Additional utilities enable saving parameters to the run node or parameter node and printing the current state Protected helpers support initializing default values and internal parameter maps This comment was generated by openai/gpt-oss-120b at temperature 0.1.

◆ set_double_param()

void PHParameterInterface::set_double_param ( const std::string & name,
const double dval )

Set a double valued parameter in the PHParameterInterface.

This routine first checks whether the interface is locked; if it is, an error message is printed and the program terminates. It then verifies that the requested parameter name exists in the default double‑parameter map. If the name is not found, a list of all implemented double parameters is printed and the function returns without modifying any value. When the name is valid, the supplied value is stored in the mutable double‑parameter map.
