Detailed Description

Reconstructs cylinder detector cells from Geant4 hits Inherits from SubsysReco and PHParameterContainerInterface to integrate with Fun4All Provides initialization per run event processing and optional energy conservation checks Configurable detector name cell geometry timing windows and output detector Maintains internal maps for binning cell size geometry nodes and cell containers Implements SetDefaultParameters to establish default reconstruction parameters Uses CheckEnergy to verify energy consistency between G4 hits and reconstructed cells This comment was generated by openai/gpt-oss-120b at temperature 0.1.

int PHG4CylinderCellReco::InitRun ( PHCompositeNode * topNode)
overridevirtual

module initialization

Initialize run‑time geometry and cell segmentation for the cylinder detector.

This method is invoked at the beginning of each new run. It locates the required DST and RUN nodes in the supplied top node, creates or retrieves the detector‑specific sub‑nodes, and obtains the hit, cell and geometry containers. It then reads the cylinder geometry, constructs the segmentation geometry container, and configures binning parameters (eta‑phi or size‑based) for each detector layer. The method also stores the resulting cell parameter container in the run node.

Parameters
topNodePointer to the top‑level PHCompositeNode of the current job.
Returns
Fun4AllReturnCodes::EVENT_OK on successful initialization; the function terminates the program with exit(1) on fatal errors. This comment was generated by openai/gpt-oss-120b at temperature 0.1.
