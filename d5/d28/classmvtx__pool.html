Detailed Description

Class mvtx_pool provides a container for decoded MVTX packets It manages loading of raw payloads linking fee identifiers to GBT links The class offers methods to add packets decode data and query various hit and trigger information Public methods include addPacket to insert a new packet Size queries for fee identifier sets and per link collections are available Accessors retrieve L1 and trigger timing counters and detector fields for a given link and bunch crossing Generic integer and long long value getters allow extraction of decoded fields by index and name Hit retrieval returns a vector of pointers to mvtx_hit objects for a specific fee identifier and strobe Verbosity control methods set and get the internal logging level A protected helper converts a fee identifier to the corresponding link identifier Private decoding routine ensures payload is processed once Helper functions load input packets and establish link mappings Internal data structures store the raw payload buffer link containers fee identifier set and verbosity flag This comment was generated by openai/gpt-oss-120b at temperature 0.1.

◆ addPacket()

int mvtx_pool::addPacket ( Packet * p)
virtual

Add a new data packet to the MVTX pool and prepare internal structures for decoding.

This method logs the call when verbosity is greater than one, validates the input pointer, resets the decoded flag, clears all GBT link buffers (preserving statistics), clears the set of fee IDs, loads the packet into the internal buffer, configures the link objects, and adjusts the buffer pointer to the payload position. If the buffer becomes empty it is cleared; otherwise any unused space is moved to the head of the buffer.

Parameters
pPointer to the Packet object to be added; may be nullptr.
Returns
Integer status code (0 for success, 0 also returned for null packet). This comment was generated by openai/gpt-oss-120b at temperature 0.1.
