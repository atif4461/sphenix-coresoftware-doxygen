Detailed Description

Class provides input handling for INTT data streams with pool management Supports streaming and triggered modes defined by InttStreamingMode Implements FillPool to load data from minimum BCO Implements CleanupUsedPackets to release packets after processing Implements CheckPoolDepth to verify sufficient data depth Implements ClearCurrentEvent to reset event state Provides GetSomeMoreEvents to request additional events based on beam clock Implements Print for diagnostic output Implements CreateDSTNode to add output node to top node tree Allows configuration of BCO range and negative BCO handling Exposes BclkStack and BeamClockFEE for external inspection Provides streamingMode setter and IsStreaming query based on run number Manages internal bookkeeping of special events run number streaming flag and packet stacks This comment was generated by openai/gpt-oss-120b at temperature 0.1.

◆ streamingMode()

void SingleInttPoolInput::streamingMode ( const bool isStreaming)

Configure INTT BCO settings based on streaming mode.

Adjusts the negative BCO offset and BCO range depending on whether streaming mode is enabled. When streaming is active, the offset is set to 96 for runs after RUN3PP_FIRST and to 97 otherwise, and the BCO range is expanded to 500. Verbosity level greater than 2 triggers an informational message. When streaming is disabled, the offset and range are reset to their minimal values.

Parameters
isStreamingtrue to enable streaming mode, false to disable it.
Returns
