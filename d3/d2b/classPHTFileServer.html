Detailed Description

TFile clean handling. It allow independant classes to access the same TFile and write ntuple to it. TFiles get written only when as many write request are achieved as open request. It get closed when the server is deleted.

open a SafeTFile. If filename is not found in the map, create a new TFile and append to the map; increment counter otherwise

Open a ROOT file using SafeTFile and manage its reference count.

This method checks whether the requested file is already present in the global SafeTFile map. If the file is already opened, the reference counter is incremented and the file’s directory is made current. If the file is not present, a new SafeTFile instance is created, inserted into the map, and its directory is set as the current one. Errors during file opening are reported to standard output.

bool PHTFileServer::write ( const std::string & filename)
static

if TFile is found in map and counter is 0, close the TFile, decrement counter otherwise

Write a ROOT file managed by PHTFileServer.

This function searches the internal SafeTFile map for the specified filename. If the file is found, it checks the reference counter to determine whether the file is still in use. When the counter is greater than one, the counter is decremented and the write operation is skipped. When the counter equals one, the file is written to disk, the counter is decremented, and a status message is printed. If the counter is zero or negative, the file is still written but a warning message is printed. If the filename is not present in the map, the function reports an error and returns false.
